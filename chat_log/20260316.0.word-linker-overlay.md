# Session: word-linker-overlay

## Goal
Build a native macOS Rust app that replaces a Chrome extension: double-click any word on screen, overlay renders a pulsing circle at that position with the word label. Foundation for drawing curved bezier lines connecting all matching instances of that word on screen.

## Current State
Working prototype committed at ~/projects/word-linker (df4af0c). Transparent fullscreen overlay renders a pulsing blue circle + word label on double-click anywhere on screen. Click-through and scroll passthrough working.

## Problem/Context
Migrating a Chrome extension that used MutationObserver + content scripts to highlight and connect matching text. Native version needs system-level input interception and screen overlay without browser DOM access.

## Solution/Approach
Three-thread architecture:
- Thread 1: CFRunLoop + CGEventTap detects double-clicks, sends coords over mpsc channel
- Thread 2: Worker receives coords, sleeps 100ms, simulates Cmd+C, reads pbpaste for selected word, writes to Arc<Mutex<ClickEvent>>
- Thread 3 (main): egui_overlay render loop, try_lock each frame, draws shapes via layer_painter

Key technical decisions:
- AX API times out (-25204) from CGEventTap callbacks and from separate threads -- appears to be a system-level issue with AXUIElementCreateSystemWide when an event tap is active
- Pasteboard approach (simulate Cmd+C, read pbpaste, restore clipboard) works universally
- Inlined egui_overlay::start() to control wgpu surface config directly
- PostMultiplied alpha mode (PreMultiplied not supported on this system)
- max_texture_dimension_2d raised to 8192 (Metal supports 16384) to fit retina framebuffer (3420x2214)

## Key Insights
- CGEventTap callback is Fn not FnMut -- Cell<T> for interior mutability of Copy types
- AX queries deadlock/timeout when called from process running a CGEventTap, even from separate threads
- GLFW on macOS always creates retina framebuffers regardless of ScaleToMonitor hint
- egui Area widget paints a background -- use layer_painter directly for transparent overlay drawing
- set_clip_rect on painter needed to prevent clipping at default egui bounds
- RxJS operators (fromEvent, merge, switchMap) map directly to Rust channel/select patterns

## Open Questions
- Why does AXUIElementCreateSystemWide timeout from a process with an active event tap? Is it a Mach port conflict?
- Would querying AX by specific PID (via NSWorkspace frontmost app) avoid the timeout?
- Best approach for finding all visible text matches -- AX tree walk vs OCR vs terminal-specific IPC?
- Should the overlay shrink to a bounding box around matches instead of fullscreen to save GPU?
