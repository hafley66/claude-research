## Key Insights

### Dioxus Native Has Legs
- Architecture is sound: Stylo (Firefox CSS) + Taffy (layout) + Vello (GPU compute renderer) + Parley (text)
- Every foundation crate is battle-tested or backed by serious projects
- Risk is the glue layer (blitz-dom, blitz-paint) maintained by 2-person team
- Even if DioxusLabs stalls, foundation crates are independent -- someone else could build new glue

### The Rendering Gap in Rust
- Chrome's efficiency = "don't draw if it didn't change" at every stage: dirty subtree layout, display item caching, composited layers, tile-based rasterization, compositor thread
- Vello is fastest raw 2D renderer (compute shaders, 100x Skia on some workloads) but ZERO incremental rendering -- full scene re-render every frame
- Missing piece: pipeline between "I know what changed" (Dioxus signals) and "I can paint fast" (Vello)
- Nobody in Rust has built dirty-tracking/caching/compositing layer yet
- A custom lightweight Dioxus renderer skipping CSS but adding Chrome-style dirty tracking is estimated 3-5k lines for basic version

### Framework Rankings
- Best reactive model: Dioxus (signals + auto-tracking)
- Best production proof: Iced (COSMIC desktop)
- Most mature: Slint (1.x stable, but GPL/commercial licensing)
- Closest to user's RxJS mental model: Floem (signal-based, but too immature)
- Fastest raw rendering: Makepad (GPU shaders, terrible docs)
- egui ceiling: O(total_widgets) per frame, ~5k+ widgets or deep nesting

### Servo is Parts Catalog, Not Competitor
- Servo = real Rust browser engine (36k stars, accelerating, alpha Summer 2026)
- Blitz cherry-picks Stylo + html5ever from Servo, uses non-Servo crates for rest
- No organizational relationship between Servo and Dioxus

### RxJS Mapping
- Signal = BehaviorSubject with auto-subscribe
- use_memo = pipe chain with distinctUntilChanged
- use_coroutine = epic/effect service
- use_resource = switchMap
- Higher-order observables (concatMap, exhaustMap) = manual coroutine work
