# Session: cc-hud-state-refactor-widget-plan

## Goal
Continue cc-hud overlay development. This session: fix inactive text color, add clickable affordance, boost scroll speed, fix chart label collisions, plan component/state architecture refactor.

## Current State
Applied 4 visual fixes (uncommitted): visible sessions all get full brightness text regardless of is_active, legend rows have subtle bg tint with brighter hover, scroll speed 4x via manual delta multiplication, chart labels inset 12px from edges to avoid y-axis tick collision. Ready to commit these, then refactor state management.

## Problem/Context
draw_big() is a 1000-line free function taking 11 parameters (7 mutable). All UI state lives as loose fields on Hud struct. Adding new features (group toggle, solo, per-row controls) keeps widening the parameter list. Need to componentize before adding more legend interactions.

## Solution/Approach
### Phase 1: State cleanup (smallest lift)
- Make draw_big() a method on Hud instead of free function. Zero extra params needed.
- Key insight: .lock().unwrap().clone() produces owned data, guard drops at semicolon, self is unborrowed after. All &mut fields freely accessible.
- No Arc<Mutex> wrapping of UI state needed yet -- just self access.

### Phase 2: UiState struct (when widgets extract)
- Bundle hidden_sessions, show_active_only, time_axis, autofit, nav_view, expanded_groups into UiState struct on Hud
- effective_hidden() becomes method on UiState
- When widgets need independent access, wrap UiState in Arc<Mutex>, lock once at top of update(), pass &mut UiState down

### Phase 3: Widget extraction
- LinkedChart: the repeated 8-chart pattern (panel_frame + draw_chart_label + base_plot + nav clamping + hover source)
- SessionLegend: ScrollArea + grouping + sorting + expand/collapse (~200 lines)
- LegendRow: already half-extracted as draw_legend_row(), convert to Widget trait
- ToolBreakdown, UsageChart, FloatingTooltip, TimeNavigator, ControlStrip

### State tiers (finalized)
- Arc<Mutex<T>> for cross-thread (poll results: HudData, UsageData)
- &mut UiState for intra-frame widget state (toggles, selections, view params)
- ctx.data_mut(id, T) for ephemeral per-frame signals (HoverState, LegendHighlight)

## Key Insights
- egui immediate mode means no subscriptions needed. "Subscription" is next frame always reading latest value.
- Borrow split: .lock().unwrap().clone() drops guard at semicolon, frees self for &mut field access
- scroll_step does not exist in egui 0.29 API. Manual delta multiplication via reading smooth_scroll_delta and adjusting ScrollArea state offset after render.
- Visible-but-not-active sessions should look the same as active in the legend. Only hidden sessions get dimmed. The axis is chart participation, not process liveness.

## Open Questions
- Should LinkedChart be a Widget impl or just a function that takes &mut UiState? Widget trait is nice but the closure-based plot API makes it awkward.
- parking_lot::Mutex vs std::sync::Mutex? Former is faster, no poisoning. Probably not worth the dep.
