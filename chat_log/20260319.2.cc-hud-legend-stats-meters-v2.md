# Session: cc-hud-legend-stats-meters-v2

## Goal
Continue building cc-hud transparent egui overlay. This session focused on: expandable session legend, usage % polling, context meter bars, stats line formatting, portable OAuth token resolution.

## Current State
Legend is scrollable with collapsible groups. Active sessions sorted to top. Context meter bars show last API call's full context (input + cache_read + cache_create) / 200k as fill level for active sessions, solid full bar for inactive. Stats line shows active session cost/context first, group total after pipe. Usage chart polls Anthropic OAuth endpoint every 90s via curl, persists to ~/.cc-hud/usage_history.jsonl. Token resolution is tiered: env var, config.toml, platform keychain. Legend hover VLines disabled (caused chart rescale).

## Key Insights
- input_tokens in JSONL usage is just the uncached sliver (often 1-3 tokens). Real context window fullness = input_tokens + cache_read_input_tokens + cache_creation_input_tokens
- Group header context should use active session specifically, not max/sum across all members
- VLines on plots affect auto_bounds, causing charts to rescale when legend is hovered. Need to either pin bounds or use a different visual indicator
- Rust doc comments (///) cannot be applied to function parameters, use // instead
