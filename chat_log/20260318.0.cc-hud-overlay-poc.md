# Session: cc-hud-overlay-poc

## Goal
Build a GPU-rendered transparent overlay HUD for Claude Code that displays turn cost/token metrics as bar charts, anchored to a tmux pane's bottom 3 lines. No ASCII rendering -- real egui/wgpu graphics.

## Current State
AC-style horizontal energy gauge bar rendering with E/F labels in rounded boxes, bronze palette. Terminal inset correction working (iTerm2 4px L/R margins). Tab detection scaffolded but disabled (AppleScript approach works but has timing/focus issues). ai-coding-tool-data skill created covering Claude Code and OpenCode data sources. Committed as 879a08b.

## Problem/Context
Detour from word-linker project. Want to visualize Claude Code session costs/tokens over time without consuming terminal real estate. The overlay sits on top of the terminal using egui_overlay (GLFW + wgpu/Metal), positioned by querying tmux pane geometry + terminal window origin via CGWindowList.

## Solution/Approach
- egui_overlay 0.9 with inlined start() for custom wgpu limits (8192 max texture for Retina)
- Three-layer position chain: tmux list-panes (cell grid) + TIOCGWINSZ (cell-to-pixel) + CGWindowList (window origin)
- TerminalInsets struct accounts for iTerm2 content margins (4px L/R, 2px top)
- below_pane_rows=1 offset to sit flush with tmux status bar
- Poll thread at 100ms updates Arc<Mutex<PixelRect>>, overlay reads each frame
- bin/launch script creates uuid-named tmux session, splits 1-line pane for daemon, attaches user
- NSApp.setActivationPolicy(.Accessory) BEFORE GLFW init to prevent focus steal / Space switch
- Move offscreen (-9999,-9999) instead of hide/show to avoid focus steal on tab toggle
- Log to /tmp/cc-hud.log for post-mortem debugging

## Key Insights
- Terminal emulators have internal content margins (iTerm2: 4px L/R, 2px top). tmux list-panes cell dimensions describe only the cell grid. tmux status bar background fills into margins. Overlay width must ADD margins to match full visual row extent.
- tmux pane height excludes the status bar row, so overlay anchored to pane bottom sits 1 row above the tmux bar. Need below_pane_rows=1 offset.
- Claude Code status line script is the best real-time data interface -- JSON on stdin after every message, zero infra needed. Better than OTel for HUD use case.
- OpenCode stores everything in SQLite (.opencode/opencode.db) -- sessions/messages/files tables, cost per session
- Copilot/Cursor have no useful local data interfaces
- GLFW show()/hide() steals focus on macOS. Move window offscreen instead.
- iTerm2 AppleScript can match tabs by tty, but tmux client tty (from list-clients) is needed, not pane tty. The client tty matches an iTerm2 session.
- egui FontFamily::Name requires the font family to have fallback fonts or text won't render at all. Double-draw with 1px offset is a working fake-bold.
- pgrep -x doesn't match macOS apps (full path in comm field). ppid walk or ps scan needed.
- Cell metrics heuristic: cell_w > 14 = Retina 2x scale
- egui::Widget for &mut T is the correct pattern for stateful widgets, not custom traits

## Open Questions
- Tab detection: AppleScript IPC is slow (~100ms per call). Should it run on its own thread with lower frequency? Or use NSWorkspace notifications instead?
- Should widgets be configurable at runtime or just TOML at startup?
- How to handle multiple monitors / different scale factors per display?
- Bold font: bundle a TTF asset or find a way to load system fonts that egui respects?
