Build an Alloy-based Rust language package (hafley-alloy) for code generation. TypeSpec models in, clean Rust out, with automatic `use` import generation via Alloy's binder/refkey system.

## Problem/Context

- This session wired symbols into components (Layer 1 -> 1.5 bridge), built SourceFile/CrateDirectory (Layer 2), and implemented cross-file reference resolution with use statement generation (Layer 3)

## Solution/Approach
Layered build, continuing from prior session:
- Layer 1 wiring: Added refkey prop + createTypeSymbol/createFunctionSymbol/createFieldSymbol calls + <Declaration symbol={sym}> + <Name /> + <Scope value={scope}> wrapping to all declaration components
- Layer 2: SourceFile wraps core SourceFile with RustSourceFileScope, CrateDirectory provides root lexical scope
- Layer 3: Reference component uses resolve() to detect cross-file refs, calls sfScope.addUse() to register imports, UseStatements reactively renders sorted `use crate::` statements
- All existing tests updated to wrap in <RustRoot> (Output + Scope with RustLexicalScope) since components now require scope context

## Key Insights
- Alloy computed() returns reactive ref with .value; memo() returns JSX children. Reference must use computed() not memo(), matching Go's pattern
- Go tests wrap in <TestPackage> for full scope chain; our equivalent is <RustRoot> with Output+Scope(RustLexicalScope)
- <Scope> wraps any { block where children might create symbols. Unit structs/semicolons skip it.
- resolve() pathDown includes RustSourceFileScope for cross-file refs, scope name = file path = module path
- pub in generated code should default true -- generated code consumers need access
- User wants serde support as future layer, common derive presets
