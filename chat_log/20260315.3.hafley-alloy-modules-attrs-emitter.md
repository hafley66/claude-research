# Session: hafley-alloy modules, attributes, emitter layer

## Goal
Build an Alloy-based Rust language package (hafley-alloy) for code generation. This session added the module system, attributes, and the first emitter layer that takes plain model definitions and outputs compilable Rust.

## Current State
- Repo: `~/projects/hafley-alloy/` on `master`
- 76 tests passing across 12 test files
- 2 cargo check validations (component-level + emitter-level)
- Depends on `@alloy-js/core@0.23.0-dev.12`
- Alloy source repo at `~/projects/alloy/` used as reference

## What Was Built

### Module System (mod.rs generation)
- `RustModuleScope`: reactive `Set<string>` tracking child module names
- `ModDirectory`: creates subdirectory + auto-generates `mod.rs` with sorted `pub mod` declarations
- `ModStatements`: renders `pub mod X;` lines from reactive set
- `SourceFile` modified: registers with parent module scope; lib.rs/mod.rs render mod declarations
- `CrateDirectory` modified: uses `RustModuleScope` instead of `RustLexicalScope`
- Nested modules work: `a/b/thing.rs` resolves to `use crate::a::b::thing::Thing`

### Attributes
- `Attributes` component: renders `#[...]` lines from string array
- `attrs` prop added to StructDeclaration, EnumDeclaration, FunctionDeclaration, StructField, TupleStructDeclaration
- Works with existing `derive` prop (derive renders first, then attrs)

### Other Component Changes
- `selfParam` prop on FunctionDeclaration: `"&"` | `"&mut"` | `"owned"` -- no more stuffing self into params array
- `externalUses` prop on SourceFile: string array rendered as `use X;` before crate uses
- FunctionParam.type made optional (for self params)

### Emitter Layer (`src/emitter/`)
- `0_types.ts`: Input schema -- ModelDef, EnumDef, ScalarType, ArrayType, MapType, ModelRef, EnumRef
- `1_type-map.tsx`: Maps neutral scalars to Rust types (String, i32, DateTime<Utc>, HashMap, Vec, Option), tracks external use deps. Returns Children (string or JSX) so model refs use refkeys.
- `2_emit-model.tsx`: Takes ModelDef/EnumDef + RefkeyRegistry, returns SourceFile JSX with derives, serde imports, snake_case filenames
- `3_emit-crate.tsx`: Pre-allocates refkey registry for all types, wraps in CrateDirectory + ModDirectory, renders
- Cross-model references work: `{ kind: "model", name: "User" }` field in BlogPost generates `use crate::models::user::User` automatically

## Key Insights
- Rust's module system requires every file to be declared by its parent (mod.rs / lib.rs) -- codegen must generate this chain
- Alloy's reactivity handles the ordering problem: child SourceFiles register after mod.rs renders, but the reactive Set triggers re-render
- RustType.code changed from string to Children so model refs can be Alloy refkeys (JSX) while scalars stay as plain strings
- Pre-allocating a RefkeyRegistry before emission lets all types reference each other regardless of declaration order
- externalUses is simpler than a full ExternCrate system -- the orchestrator already knows what deps each file needs
