## Goal
Continue cc-hud overlay development. This session covered: fixing inactive text color, adding eye icons for visibility toggle, group toggle, architecture planning for state refactor and widget extraction.

## Current State

Eye icons work on all three row types (flat, group header, sub-row). Group header has split click zones: eye zone toggles all members, main zone expands/collapses. ## Problem/Context
draw_big() is a 1000+ line free function with 11 params (7 mutable). Architecture discussion established a refactor path: make it a method on Hud, then extract UiState struct, then extract widgets. The borrow-split insight: .lock().unwrap().clone() drops guard at semicolon, freeing self for &mut field access.

## Key Insights
- show_active_only is a mask, not a filter. It force-hides inactive sessions on top of manual hidden_sessions set. Users opt individual inactive sessions back in.
- Borrow split: .lock().unwrap().clone() produces owned data, guard drops at semicolon, self unborrowed after. All &mut fields freely accessible.
- egui immediate mode means no subscriptions needed. Next frame always reads latest value.
- scroll_step does not exist in egui 0.29. Manual delta multiplication via smooth_scroll_delta + ScrollArea state offset.
- React-query mapping: HudData polling = useQuery with refetchInterval, build_chart_data = useMemo, hidden_sessions = useState, ctx.data_mut = BehaviorSubject
