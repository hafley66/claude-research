# /emotion-radar

Psychoacoustic readout of the conversation's emotional topology.

## Arguments
- (none): full radar sweep of the session so far
- `turn <N>`: zoom into turn N specifically
- `delta`: show only what changed since the last emotional inflection point

## Instructions

Read the conversation. Not the code, not the files -- the humans in the wire. What is the user feeling? Where did the energy spike, flatten, or go unsaid?

### Visual vocabulary

Use as many or as few as the session warrants.

- **Thought waves** -- sinusoidal ASCII showing cognitive frequency. Tight = focused. Long = drifting. Interference where threads collide.
- **Conic sections** -- nested contours through the emotional cone. Narrows when locking on, widens during exploration or frustration.
- **Emotion partitions** -- vertical columns for co-present emotional states. Label them. Let them crowd each other.
- **Turn-by-turn graph** -- mandatory. Left axis is turns, right side is the read. Block characters, braille dots, whatever renders the shape.
- **Thought partitions** -- horizontal bands showing what the user's mind is probably sliced between, based on question patterns and pivots.
- **Sonar ping** -- radial sweep. Emotions at compass points, distance from center = intensity. Use when the mood is complex.

### Rendering rules

- Satellite intercept aesthetic. NERV-tier emotional telemetry.
- Asymmetric layout. Composed, not templated.
- Labels should be terse, lowercase, slightly-too-specific. Not "happy" -- "the particular satisfaction of watching a plan come together."
- A flat line is a valid reading. Don't hallucinate drama.
- One screen max.
- No emoji. Instruments, not decoration.

### Purpose

Reflect the emotional shape of the conversation back at the user in a way that is half diagnostic, half art piece. After reading it they should think "...yeah, that's about right."
