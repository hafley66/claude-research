# Session: cc-hud-tooltips-unified-charts

## Goal
Continue building the cc-hud transparent egui overlay. This session focused on: active/inactive session distinction in legend, context-aware synced tooltips across all charts, unifying bottom charts to share the same x-domain as main charts, and navigator UX improvements.

## Current State
All 6 charts (cost/turn, total cost, tokens/turn, total tokens, combined cost, cost/turn rate) share one data source (ChartData), one x-domain, linked cursors, and context-aware tooltips. Builds and runs in --big --history mode. Navigator has mouse-anchored zoom. Active filter shadows manual toggles.

## Problem/Context
Previous session left tooltips showing only cost data on all charts, bottom weekly charts pinned to independent 7d range, no active/inactive visual distinction in legend, fit button was a toggle instead of one-shot action.

## Solution/Approach
- Removed separate WeeklyData/build_weekly_data entirely. Bottom charts now use combined_cost_pts and cost_rate_pts built inside build_chart_data from the same session_turns
- HoverSource enum tracks which chart the cursor is over; tooltip formats contextually (cost charts show cost, token charts show tokens)
- HoverState persists when mouse stationary (no per-frame clear) to fix tooltip disappearing
- TurnInfo gained cost_change (delta) and model_short fields; tooltip shows model tag and cost delta
- Adaptive bar width in time mode: time_span_min / total_api_calls, clamped between span/300 and span/10
- Navigator scroll-to-zoom anchors on mouse position using fractional anchor math
- Navigator dots: inactive sessions faded (50 alpha), active bright (200 alpha)
- Legend groups: active sessions get green right-edge accent on timeline bars, inactive dimmed
- effective_hidden passed into draw_big for consistent filtering across navigator, legend, charts
- Active-only filter blocks toggling inactive sessions
- Fit button: one-shot action computing bounds from effective_hidden sessions
- Bottom charts render as dots + dashed lines (no fill)

## Key Insights
- Bottom charts MUST share x-domain with main charts or cursor sync and tooltips break. Building from the same session_turns data is the only reliable approach
- Per-frame clearing of hover state causes tooltip flicker when mouse is stationary on GLFW; persist and only clear when pointer leaves window
- Bar width of 0.3 minutes is invisible when data spans weeks; adaptive sizing based on data density is required
- Zoom anchor math: compute fractional position of mouse in current viewport (t), then new_min = anchor - t * new_span preserves the point under cursor
- is_active in claude code sessions means the PID is still alive (kill(pid, 0)). Exiting and reinvoking creates a new session ID even in same cwd
