# Session: cc-hud-data-charts

## Goal
Wire real Claude Code usage data into the cc-hud overlay, build multi-session charts showing cost, tokens, tool use, and agent spawns. Evolving toward a full local usage analytics dashboard.

## Current State
Working overlay with three panels: per-turn cost diverging bars (input cost up, output cost down), per-turn token diverging bars, and session legend with cost split. Reads JSONL transcripts directly from ~/.claude/projects/ -- zero config needed. --history flag scans all historical sessions. Builds and runs.

## Problem/Context
Previous session built the transparent overlay POC with tmux pane anchoring and AC6 gauge bar. This session pivoted from the gauge to real data: multi-session charting from CC's JSONL transcripts. Key discovery: JSONL files contain all needed data (tokens, model, tool uses) without any status line config override.

## Solution/Approach
- Data layer scans ~/.claude/sessions/*.json for active PIDs, tails each session's JSONL
- Parses assistant messages for usage (input/output/cache tokens), tool_use content blocks, Agent spawns
- Computes cost from model-specific pricing (Opus 4.6=$5/$25, Sonnet=$3/$15, etc.)
- Cost split into input cost (base input + cache read + cache write) vs output cost
- --history flag does full scan of all ~/.claude/projects/*/*.jsonl on first tick
- Three-panel HUD: cost bars (diverging), token bars (diverging), legend with cost split
- Status line script also available at bin/cc-hud-status.sh but JSONL approach preferred (non-invasive)

## Key Insights
- JSONL transcripts are the zero-config path for usage dashboards; status line requires overriding existing display
- Subagent costs NOT separately tracked: caller field is always {"type":"direct"}, no way to attribute cost to Agent tool_use
- Cumulative cost lines are useless for ratio visualization -- both monotonically increase together. Per-turn diverging bars show the actual ratio
- Opus 4.6/4.5 pricing: $5 in, $25 out. Cache read = 0.1x, cache write 5m = 1.25x. JSONL doesn't distinguish cache tier
- Message types in JSONL: file-history-snapshot, user, assistant, progress, system, queue-operation, last-prompt
- Tool names available: Glob, Read, Agent, Edit, Bash, Skill, Grep, Write
- Active sessions discoverable via ~/.claude/sessions/<pid>.json + kill -0 check
- Project dir naming: cwd path with / replaced by - (e.g. /Users/chris/projects/foo -> -Users-chris-projects-foo)
